免費使用
LINE Messaging API 免費方案每月提供 200 則推播訊息。本專案每月約使用 20 則（週一至週五各 1 則），完全在免費範圍內，無需付費。

傳送範圍

個人通知
支援
傳送到你個人的 LINE 帳號，設定 LINE_USER_ID 即可。
群組通知
支援
傳送到指定 LINE 群組，設定 LINE_GROUP_ID 即可。
同時傳送
支援
兩個 Secret 都設定時，個人與群組會同時收到通知。

Step 1 — 取得 LINE_CHANNEL_ACCESS_TOKEN

1
前往 LINE Developers Console

用 LINE 帳號登入 developers.line.biz/console

2
建立 Provider

點 Create a new provider，名稱隨意填（例如「Investment Bot」）

3
建立 Messaging API Channel

在 Provider 下點 Create a new channel → 選 Messaging API → 填寫基本資料

4
產生 Channel access token

進入 channel → 切換到 Messaging API 分頁 → 拉到最下方 → 點 Issue → 複製 token

Step 2 — 取得 LINE_USER_ID（個人通知）

1
掃描 QR Code 加 Bot 為好友

在 channel 頁面的 Messaging API 分頁找到 QR Code，用 LINE 掃描加入

2
查看 Your user ID

回到 channel → Basic settings 分頁 → 拉到最下方 → 複製 Your user ID

U開頭，例如 Uabcd1234efgh5678

Step 3 — 取得 LINE_GROUP_ID（群組通知，可選）

1
將 Bot 加入目標群組

在 LINE 群組中邀請剛建立的 Bot 加入

2
在群組對 Bot 傳任意訊息

例如傳「hi」，觸發 webhook 事件

3
用 webhook.site 接收 Group ID

前往 webhook.site 取得一個臨時 URL → 在 LINE Developers Console 的 Messaging API 分頁設定為 Webhook URL → 重新在群組傳一則訊息 → 在 webhook.site 看到的 JSON 中找 groupId 欄位

C開頭，例如 Cabcd1234efgh5678
注意：Bot 加入群組後，需在 LINE Developers Console 的 Messaging API 分頁將 Allow bot to join group chats 設為 Enabled。

Step 4 — 設定 GitHub Secrets

GitHub repo → Settings → Secrets and variables → Actions → New repository secret

Secret 名稱 說明 必填
LINE_CHANNEL_ACCESS_TOKEN Bot 的 Channel access token
Step 1 取得的長字串 token		 ✅ 必填
LINE_USER_ID 個人 LINE User ID
Step 2 取得，U 開頭		 可選
LINE_GROUP_ID 群組 ID
Step 3 取得，C 開頭		 可選
