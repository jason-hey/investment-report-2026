台股當日選股訊號規劃

在每日報告中找出「當天最有上漲機率」台股的資料規劃（2026-07-03）
⚠️
前提：機率傾斜，不是預測
沒有任何資料能「預測」當天必漲。這份規劃做的是把幾類統計上有領先性的訊號疊加起來，篩出上漲機率相對高的觀察清單。 報告在台灣時間 08:00 生成、台股 09:00 開盤 — 這個時間差是關鍵優勢：有一整批「隔夜領先指標」在生成當下已經可以取得。

一、隔夜領先訊號（08:00 已可取得，對「當天」最有預測力）

最值得優先加入的一區 — 這些訊號直接領先當天開盤。
台積電 ADR（TSM）漲跌與對 2330 溢價率 領先性 ★★★
台股單一最強的開盤方向指標。ADR 大漲且溢價擴大，2330 與權值股高機率開高。同理可追蹤 UMC（聯電）、ASX（日月光）ADR。 頁面上應顯示：ADR 收盤漲跌%、換算台幣後對 2330 現股的溢價率、近 20 日溢價率區間對照。
資料來源：yfinance（TSM / UMC / ASX + 匯率 TWD=X），Python 直接計算溢價率
台指期夜盤 → 現貨開盤缺口換算 領先性 ★★★
報告已有夜盤區塊，建議升級：不只列數字，而是換算成「對應現貨預期開盤缺口（點數 / %）」，並標示大盤開高開低對個股觀察清單的意義。
資料來源：期交所（TAIFEX）夜盤行情，API 預抓取代 web_search
美股族群表現 → 台股供應鏈對照表 領先性 ★★★
維護一張固定的映射表：NVDA / SOX 大漲 → AI 伺服器台鏈（緯創、廣達、奇鋐、雙鴻…）；AAPL 漲 → 蘋概股（鴻海、大立光…）； MU / HBM 漲 → 記憶體族群（南亞科、群聯…）。每天用美股實際漲跌自動點亮對應的台股受惠族群。 這是報告最能發揮價值的地方 — 映射表寫死在程式裡，只有漲跌數據每天更新。
資料來源：yfinance（美股個股 + ^SOX），族群映射表維護在 repo

二、籌碼面（前一日收盤資料，勝率穩定）

報告已有法人連三日買賣超，以下為建議新增項目。
外資 + 投信「同步買超」清單（土洋合買） 領先性 ★★
兩大法人同日（或連續數日）同步買超的個股，後續勝率比單一法人買超高。可與現有的連三日排行整合成同一區塊。
資料來源：TWSE OpenAPI（openapi.twse.com.tw）三大法人買賣超，Python 交集計算
買超金額佔成交值比重（取代絕對金額排序） 領先性 ★★
買超 5 億對大型股沒感覺，對中型股就是強訊號。排行榜應改用「買超金額 ÷ 當日成交值」排序，才能真正找出籌碼被鎖定的股票。
資料來源：TWSE OpenAPI 法人買賣超 + 個股成交值，Python 計算
軋空候選股：券資比偏高 + 股價轉強 領先性 ★★
融券餘額高（券資比 > 30%）且股價開始上漲的股票，有融券強制回補買盤推升。搭配「借券賣出餘額大幅回補」（空方撤退訊號）一起看。
資料來源：TWSE 融資融券餘額 API、借券賣出餘額

三、事件催化（可預先知道日期的利多）

月營收公布：創新高 / YoY 大增清單 領先性 ★★
每月 10 日前公布。營收創歷史新高或 YoY 大幅成長的股票，公布當天與隔天有明顯超額報酬。營收公布旺日（每月 7–10 日）這區塊特別有價值。
資料來源：公開資訊觀測站（MOPS）營收彙總表，可自動抓取
台股法說會日程 領先性 ★
法說會前後常有行情。報告已有美股財報日曆，加台股法說會就是同一種區塊的延伸。
資料來源：MOPS 法說會公告
庫藏股公告、除權息前卡位 領先性 ★
庫藏股買回期間有公司自己的買盤支撐；高殖利率股除息前常有卡位行情。
資料來源：MOPS 重大訊息、TWSE 除權息預告表

四、技術面（輔助過濾用）

量價齊揚突破清單 領先性 ★
突破 20 日新高、且成交量 > 5 日均量 1.5 倍的個股清單。單獨用勝率普通，與籌碼訊號疊加後價值明顯提升。
資料來源：yfinance .TW 日線，Python 計算，不需 AI 搜尋
相對強度（RS）排名 領先性 ★
近 20 日漲幅減去加權指數漲幅的排名。「強者恆強」在台股中小型股特別明顯，適合當作綜合評分的其中一欄。
資料來源：yfinance 日線 + 加權指數（^TWII），Python 計算

呈現方式：「今日觀察清單」綜合評分表

與其各區塊分開列，更有用的做法是做一張綜合評分表：每檔股票列出命中哪些訊號，命中越多排越前面，並附一行「為什麼」。示意如下：
股票ADR/族群連動土洋合買營收創高量價突破軋空候選得分理由摘要
0000 範例A✓✓✓✓—4 NVDA 大漲 + 外資投信連 3 日同買 + 6 月營收創高
0000 範例B✓✓—✓—3 SOX 上漲受惠 + 買超佔成交值 8%
0000 範例C—✓—✓✓3 券資比 42% 開始轉強，軋空題材

實作原則（三個提醒）

1
全部用 API 預抓、Python 計算，再注入報告。 正好呼應改善分析文件的 #1（模板 + JSON 分離）和 #4（TWSE API 預抓）。讓 AI 用 web_search 去「找」這些數字，準確率會毀掉整個功能的價值。
2
一定要做勝率回顧區塊。 報告應顯示「昨天觀察清單的股票，今天實際表現如何」並累積歷史勝率統計。沒有這個回饋機制，永遠不知道這套訊號到底有沒有用，也無法淘汰沒用的訊號。
3
免費穩定的資料來源已足夠。 TWSE OpenAPI（openapi.twse.com.tw）、TPEx、MOPS 公開資訊觀測站、yfinance 幾乎涵蓋上述所有籌碼、營收、價量資料，不需付費資料源即可起步。
